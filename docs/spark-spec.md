# Spark — Frontend Specification
**Версия:** 1.1  
**Дата:** март 2026  
**Стек:** React, YouTube IFrame API, Tailwind CSS, React Router, Shadcn UI

---

## 1. Обзор продукта

**Spark** — мобильное веб-приложение для изучения английского через YouTube-контент. Пользователь создаёт плейлисты, добавляет в них фрагменты с YouTube, задаёт временной интервал и описание. Фрагменты в плейлисте воспроизводятся последовательно — по окончании одного автоматически запускается следующее. Главный экран — горизонтальные полочки плейлистов.

**Целевая аудитория:** люди которые самостоятельно учат английский и хотят организовать учебные материалы по своим темам.

**Платформа:** мобильный браузер (целевой viewport 390×844).

---

## 2. Стек и окружение

- **Фреймворк:** React 19
- **Плеер:** YouTube IFrame API (`window.YT.Player`)
- **Хранилище:** `localStorage` (персистентность перманентная)
- **Стили:** Tailwind CSS
- **Шрифты:** Syne (заголовки), DM Sans (текст)
- **Цветовая схема:** поддержка тёмной и светлой тем с возможностью ручного переключения в шапке. Состояние темы сохраняется в `localStorage`.

---

## 3. Структура данных

### Плейлист (`Playlist`)

```ts
{
  uuid: string
  name: string 
  createdAt: number
  fragmentIds: string[]
}
```

### Фрагмент (`Fragment`)

```ts
{
  uuid: string
  youtubeId: string
  title: string
  description: string
  isVertical: boolean // узнаём из ссылки на видео по path (*/shorts/*)
  timeStart: string|null
  timeEnd: string|null
  createdAt: string
}
```

Две вкладки в нижнем таббаре.

| Tab | Иконка (Lucide) | Лейбл | Компонент |
|---|---|---|---|
| `playlists` | `LayoutGrid` | Плейлисты | `PlaylistsPage` |

## 5. Экран: Плейлисты (`PlaylistsPage`)

**Маршрут:** `playlists`  
**Назначение:** обзор всех созданных пользователем плейлистов в виде горизонтальных полочек

### Структура экрана

Вертикальный скролл. Каждый плейлист — отдельная «полочка»: заголовок ряда и горизонтальный скролл карточек фрагментов.

**Шапка полочки:**
- Название плейлиста слева, открывается по клику «все →» или по клику на название плейлиста  
- Счётчик фрагментов (напр. `8`)
- Кнопка действий справа: `все →` (переход к деталям плейлиста, текстовая кнопка). 

**Горизонтальный ряд карточек:**
- `overflow-x: scroll`, `scroll-snap-type: x mandatory`, скроллбар скрыт
- Карточки фиксированной ширины (~110px), `scroll-snap-align: start`

**Кнопка «Создать плейлист»** — находится в шапке (PageHeader) в виде иконки `Plus`. Открывает страницу `/add` для создания плейлиста.

### Пустое состояние

Если плейлистов нет — по центру экрана иконка ✦, текст «Пока нет плейлистов», подпись «Создайте первый и добавьте фрагмент», кнопка «Создать плейлист».

### Порядок плейлистов

Плейлисты отображаются в порядке создания (новые сверху).

---

## 6. Компонент: Карточка фрагмента в полочке (`ShelfCard`)

Ширина ~110px, фиксированная. Состоит из превью и подписи.

**Превью:**
- Соотношение сторон 16:9 (`aspect-video`)
- Фоновое изображение: `https://img.youtube.com/vi/{youtubeId}/mqdefault.jpg`

**Подпись:**
- Название фрагмента, 2 строки максимум (`-webkit-line-clamp: 2`)

**Клик по карточке** — переход на экран фрагмента (`VideoPage` / `FragmentPage`).

## 7. Экран: Плейлист (`PlaylistDetailsPage`)

**Маршрут:** `playlist/:playlistId`
**Назначение:** полный список фрагментов одного плейлиста + управление плейлистом

### Шапка

Кнопка «←» назад. Название плейлиста. Кнопка `⋯` (три точки) — меню: редактировать плейлист (предзаполненный `prompt` для имени), удалить плейлист (модальное окно для подтверждения).

### Список фрагментов

Вертикальный список. Каждый элемент — компонент `ListItem`:
- Превью 72×48px (скруглённые углы, без рамки)
- Название (одна строка, overflow ellipsis)
- **Индикатор фрагмента**: яркий бейдж с иконкой `Scissors` (Lucide) + временной интервал (напр. `0:30 – 1:45`). Оформлен с фоном `accent/10` и жирным шрифтом.

Длинный тап или свайп влево по элементу — показывает кнопку удаления фрагмента из плейлиста.

### Управление

**Кнопка воспроизведения** — крупная круглая кнопка `Play` справа от названия плейлиста. Запускает первый фрагмент.

**Кнопка добавления** — находится в шапке (PageHeader) рядом с меню действий. Переход на `AddPage` с предзаполненным `playlistId`.

### Пустое состояние

Если фрагментов в плейлисте нет — иконка плейлиста по центру, текст «Плейлист пуст», кнопка «Добавить первый фрагмент».

---

## 8. Экран: Просмотр фрагмента (`FragmentPage`)

**Маршрут:** открывается по клику на карточку фрагмента  
**Назначение:** воспроизведение фрагмента + просмотр описания

### Структура

**Плеер** занимает верхнюю часть экрана. Высота зависит от ориентации:
- Горизонтальное (`isVertical: false`) — `padding-top: 56.25%` (16:9)
- Вертикальное (`isVertical: true`) — `padding-top: 100%` (1:1)

**Под плеером — панель информации** (скроллится):
- Название фрагмента (берём через YouTube oEmbed API)
- Тоггл «Зациклить плейлист» в информационной панели под описанием (состояние сохраняется в `localStorage`). Выполнен в виде современного ползунка (Switch) с чёткой индикацией состояния (акцентный цвет при включении, видимая граница при выключении).
- Бейдж `Scissors` + `timeStart – timeEnd` если задан диапазон
- Описание (свёрнуто до 110 символов, кнопка «ещё»)
- Кнопка «Редактировать» (Lucide `Edit`) — переход на `EditPage`

### Поведение плеера (YT IFrame API)

**Инициализация:**

`YT.Player` создаётся после готовности `window.YT`. Параметры:

```
autoplay: 1
controls: 1
modestbranding: 1
rel: 0
showinfo: 0
iv_load_policy: 3
mute: 1  // обязательно для autoplay в браузере
start: timeStart (если задан)
```

При `onReady` → `playVideo()`.

**Автовоспроизведение следующего фрагмента в плейлисте:**

Маршрут `/video/:playlistId/:videoId` передаёт контекст плейлиста в `FragmentPage`. При открытии плеер загружает упорядоченный список фрагментов плейлиста.

Переход к следующему фрагменту (`goToNext`) происходит в двух случаях:

- **`onStateChange: ENDED`** — фрагмент досмотрен до конца (для фрагментов без ограничения по времени или если пользователь досмотрел дальше `timeEnd`)
- **Достижение `timeEnd`** — если задан временной интервал, `setInterval` (каждые 100ms) проверяет позицию воспроизведения; при `currentTime >= timeEnd - 0.15` вызывается `goToNext`

```
function goToNext():
  currentIndex = playlistFragments.indexOf(currentFragment)
  nextFragment = playlistFragments[currentIndex + 1]
  if nextFragment:
    navigate(/video/{playlistId}/{nextFragment.uuid})
  else if isLooping:
    // Если включено зацикливание — возвращаемся к первому фрагменту
    navigate(/video/{playlistId}/{playlistFragments[0].uuid})
  else:
    navigate(/playlist/{playlistId})
```

Интервал запускается при `PLAYING` и очищается при `PAUSED / BUFFERING / ENDED` и при анмаунте компонента.

Если текущий фрагмент последний в плейлисте — происходит возврат на страницу плейлиста.

**Очистка:**

`cancelAnimationFrame(rafId)` + `player.destroy()` при анмаунте компонента.

**Состояния плеера:**

- `BUFFERING (3)` → спиннер в центре поверх оверлея
- `PLAYING (1)` → спиннер скрыт, иконка паузы скрыта
- `PAUSED (2)` → иконка ▶ в центре

## 9. Создание плейлиста (объединено с добавлением фрагмента)

Пустые плейлисты не допускаются. Создание плейлиста всегда требует добавления первого фрагмента.

**Маршрут:** `/add` (без параметра `playlistId`)

Кнопка «Создать плейлист» в `PlaylistsPage` ведёт на `/add` (Link, не модал).

### Форма

Форма состоит из двух секций, разделённых визуальным разделителем:

**Секция 1 — Плейлист:**
- Название плейлиста — text input, placeholder «Мой плейлист», autofocus

**Секция 2 — Первый фрагмент:**
- YouTube URL — text input (парсинг `youtubeId` через regex, debounce 500ms)
- Превью — thumbnail при валидном `youtubeId`
- Название фрагмента — text input, автозаполнение через YouTube oEmbed API
- Описание — textarea (необязательно)
- Временной диапазон — toggle + 2 inputs (необязательно)

### Валидация фрагмента через iframe

При нажатии «Создать плейлист»:
1. Валидация полей (название плейлиста, URL, диапазон)
2. Создаётся скрытый `YT.Player` (1×1px, off-screen) с указанным `youtubeId`
3. `onReady` → фрагмент доступен → сохраняем плейлист + фрагмент → переход на `/playlists`
4. `onError` → ошибка «Видео недоступно или указан неправильный URL»
5. Таймаут 8 секунд → ошибка (фрагмент недоступен)

Кнопка во время проверки: disabled, спиннер + текст «Проверяем фрагмент…»

## 10. Экран: Добавление фрагмента в существующий плейлист (`AddPage`)

**Маршрут:** `/add?playlistId={uuid}`
**Назначение:** добавление нового фрагмента в существующий плейлист

Та же форма что и при создании, но без секции «Название плейлиста». Вместо неё — select/дропдаун для выбора плейлиста (предзаполнен если передан `playlistId`).

Фрагмент также проходит валидацию через скрытый iframe перед сохранением.

### Поля формы

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| YouTube URL | text input | да | Из URL парсится `youtubeId` через regex `(?:v=\|youtu\.be\/|shorts\/)([a-zA-Z0-9_-]{11})` |
| Плейлист | select / выбор | да | Дропдаун из существующих плейлистов. Если открыт из конкретного плейлиста — предзаполнен. |
| Описание | textarea | нет | — |
| Временной диапазон | toggle + 2 inputs | нет | Активируется отдельным тоглом |
| Начало фрагмента | text input | нет | Формат `m:ss`, placeholder `0:30` |
| Конец фрагмента | text input | нет | Формат `m:ss`, placeholder `1:45` |

### Превью при вводе URL

Как только `youtubeId` успешно извлечён — показывается превью: `https://img.youtube.com/vi/{youtubeId}/hqdefault.jpg`. Скруглённые углы, акцентная граница. Если `youtubeId` не удалось распарсить — превью не показывается, под полем URL текст «Не удалось распознать ссылку» (появляется через 500ms после ввода, debounce).

### Валидация

| Поле | Правило | Сообщение |
|---|---|---|
| YouTube URL | Должен распарситься `youtubeId` | «Введите корректную ссылку» |
| Плейлист | Выбран | «Выберите плейлист» |
| timeStart / timeEnd | Оба заполнены если включён диапазон | «Заполните оба поля» |
| timeEnd > timeStart | Числовая проверка | «Конец должен быть позже начала» |
| Доступность фрагмента | Проверка через скрытый iframe | «Видео недоступно или указан неправильный URL» |

Ошибки — inline, под соответствующим полем. Кнопка «Сохранить» активна всегда, ошибки показываются при сабмите.

### После сохранения

Фрагмент добавляется в выбранный плейлист (`playlist.fragmentIds`), данные сохраняются в `localStorage`. Переход на главный экран, плейлист которому добавили фрагмент — прокручен в начало, новая карточка видима.

## 11. Экран: Редактирование фрагмента (`EditPage`)

Идентичен `AddPage` по структуре, но все поля предзаполнены данными фрагмента. Дополнительно — кнопка «Удалить фрагмент» внизу (красный цвет, требует подтверждения через алерт/модал «Удалить?»).

После сохранения — возврат на `PlaylistDetailsPage` с обновлёнными данными.

---

## 12. Хранилище (`localStorage`)

Все данные хранятся в двух ключах:

```
spark_playlists  →  Playlist[]   (JSON)
spark_fragments  →  Fragment[]   (JSON, все фрагменты всех плейлистов)
```

**Запись:** при каждом изменении (`addFragment`, `editFragment`, `deleteFragment`, `createPlaylist`, `editPlaylist`, `deletePlaylist`).

**Чтение:** при инициализации приложения (`App` useEffect).

**Обработка ошибок:** `try/catch` при парсинге. Если данные повреждены — инициализация с пустыми массивами, данные не перезаписываются автоматически (нужно явное действие пользователя).

## 15. Известные ограничения и точки роста v2

**Порядок фрагментов в плейлисте** — в v1 сортировка по дате создания (`createdAt`). В v2 — drag-and-drop сортировка.

**Синхронизация между устройствами** — в v1 только `localStorage`. В v2 — облачный бэкенд.

**Режим изучения** — отдельный режим флэшкарточек по словам плейлиста. В v2.

**Autoplay и mute** — браузеры блокируют autoplay со звуком без пользовательского жеста. Первое воспроизведение фрагмента всегда без звука.
