# Spark — Frontend Specification
**Версия:** 1.3  
**Дата:** март 2026  
**Стек:** React, YouTube IFrame API, Tailwind CSS, React Router, Shadcn UI

---

## 1. Обзор продукта

**Spark** — мобильное веб-приложение для изучения английского через YouTube-контент. Пользователь создаёт сегментированные видео, добавляет в них сегменты с YouTube, задаёт временной интервал и описание. Сегменты в сегментированном видео воспроизводятся последовательно — по окончании одного автоматически запускается следующее. Главный экран — горизонтальные полочки сегментированных видео.

**Целевая аудитория:** люди которые самостоятельно учат английский и хотят организовать учебные материалы по своим темам.

**Платформа:** мобильный браузер (целевой viewport 390×844).

---

## 2. Стек и окружение

- **Фреймворк:** React 19
- **Плеер:** YouTube IFrame API (`window.YT.Player`)
- **Хранилище:** `localStorage` (персистентность перманентная)
- **Стили:** Tailwind CSS
- **Шрифты:** Syne (заголовки), DM Sans (текст)
- **Цветовая схема:** поддержка тёмной и светлой тем с возможностью ручного переключения в шапке. Состояние темы сохраняется в `localStorage`.

## 3. Структура данных

### Сегментированное видео (`SegmentedVideo`)

```ts
{
  uuid: Uuid

  createdAt: Timestamp
  segmentIds: string[]
}
```

### Видео (`Video`)

```ts
{
  uuid: string
  youtubeId: string
  description: string
  duration: number
  isEmbeddable: boolean
  isVertical: boolean
  createdAt: Timestamp
}
```

### Сегмент (`Segment`)

```ts
{
  uuid: Uuid
  description: string
  timeStart: string
  timeEnd: string
  video: Video
  createdAt: Timestamp
}
```

## 5. Экран: Сегментированные видео (`SegmentedVideosPage`)

**Маршрут:** `segmented-videos`  
**Назначение:** обзор всех созданных пользователем сегментированных видео в виде горизонтальных полочек

### Структура экрана

Вертикальный скролл. Каждое сегментированное видео — отдельная «полочка»: заголовок ряда и горизонтальный скролл карточек сегментов.

**Шапка полочки:**
- Название сегментированного видео слева, открывается по клику «все →» или по клику на название сегментированного видео  
- Счётчик сегментов (напр. `8`)
- Кнопка действий справа: `все →` (переход к деталям сегментированного видео, текстовая кнопка). 

**Горизонтальный ряд карточек:**
- `overflow-x: scroll`, `scroll-snap-type: x mandatory`, скроллбар скрыт
- Карточки фиксированной ширины (~110px), `scroll-snap-align: start`

**Кнопка «Создать сегментированное видео»** — находится в шапке (PageHeader) в виде иконки `Plus`. Открывает страницу `/add` для создания сегментированного видео.

### Пустое состояние

Если сегментированных видео нет — по центру экрана иконка ✦, текст «Пока нет сегментированных видео», подпись «Создайте первое и добавьте сегмент», кнопка «Создать сегментированное видео».

### Порядок сегментированных видео

Сегментированные видео отображаются в порядке создания (новые сверху).

## 6. Компонент: Карточка сегмента в полочке (`ShelfCard`)

Ширина ~110px, фиксированная. Состоит из превью и подписи.

**Превью:**
- Соотношение сторон 16:9 (`aspect-video`)
- Фоновое изображение: `https://img.youtube.com/vi/{youtubeId}/mqdefault.jpg`

**Карточка:**
- Превью видео

**Клик по карточке** — переход на экран сегмента (`SegmentPage`).

## 7. Экран: Сегментированное видео (`SegmentedVideoPage`)

**Маршрут:** `segmented-video/:segmentedVideoId`
**Назначение:** полный список сегментов одного сегментированного видео + управление сегментированным видео

### Шапка

Кнопка «←» назад. Название сегментированного видео. Кнопка `⋯` (три точки) — меню: удалить сегментированное видео (модальное окно для подтверждения).

### Список сегментов

Вертикальный список. Каждый элемент — компонент `ListItem`:
- Превью 72×48px (скруглённые углы, без рамки)
- Описание (две строки, overflow ellipsis)
- **Индикатор сегмента**: яркий бейдж с иконкой `Scissors` (Lucide) + временной интервал (напр. `0:30 – 1:45`). Оформлен с фоном `accent/10` и жирным шрифтом.

### Управление

**Кнопка воспроизведения** — крупная круглая кнопка `Play` справа от названия сегментированного видео. Запускает первый сегмент.

**Кнопка добавления** — находится в шапке (PageHeader) рядом с меню действий. Переход на `AddPage` с предзаполненным `segmentedVideoId`.

### Пустое состояние

Если сегментов в сегментированном видео нет — иконка сегментированного видео по центру, текст «Сегментированное видео пусто», кнопка «Добавить первый сегмент».

## 8. Экран: Просмотр сегмента (`SegmentPage`)

**Маршрут:** открывается по клику на карточку сегмента  
**Назначение:** воспроизведение сегмента + просмотр описания

### Структура

**Плеер** занимает верхнюю часть экрана. Высота зависит от ориентации:
- Горизонтальное (`isVertical: false`) — `padding-top: 56.25%` (16:9)
- Вертикальное (`isVertical: true`) — `padding-top: 100%` (1:1)

**Под плеером — панель информации** (скроллится):
- Описание сегмента (то, что ввёл пользователь)
- Тоггл «Зациклить сегментированное видео» в информационной панели под описанием (состояние сохраняется в `localStorage`). Выполнен в виде современного ползунка (Switch) с чёткой индикацией состояния (акцентный цвет при включении, видимая граница при выключении).
- Бейдж `Scissors` + `timeStart – timeEnd` если задан диапазон
- Описание — реализовано через компонент `ExpandableDescription` (свёрнуто до 110 символов, кнопка «Развернуть/Свернуть»).
- Кнопка «Редактировать» (Lucide `Edit`) — переход на `EditPage`

### Поведение плеера (YT IFrame API)

**Инициализация:**

`YT.Player` создаётся после готовности `window.YT`. Параметры:

```
autoplay: 1
controls: 0  // нативные контролы отключены (кастомные реализованы отдельно)
modestbranding: 1
rel: 0
showinfo: 0
iv_load_policy: 3
mute: 1  // обязательно для autoplay в браузере
start: timeStart (если задан)
```

При `onReady` → `playVideo()`.

**Кастомные контролы (`PlayerControls`):**

Реализованы через хук `usePlayerControls`, который каждые 250мс читает состояние плеера через YouTube IFrame API. Состояние `isPlaying` берётся напрямую из `player.getPlayerState()` — никогда не изменяется оптимистично при нажатии кнопок.

Контролы:
- **Play/Pause** — вызывает `playVideo()` / `pauseVideo()` в зависимости от состояния
- **Прогресс-бар** — 0–100% соответствует диапазону `timeStart–timeEnd`; клик пересчитывает позицию относительно отрезка и вызывает `seekTo`
- **Отображение времени** — `currentTime / duration` в формате `m:ss`, оба значения относительны к отрезку
- **Громкость** — слайдер 0–100, вызывает `setVolume`
- **Mute/Unmute** — вызывает `mute()` / `unMute()`; состояние сохраняется в `sessionStorage`
- **Скорость воспроизведения** — кнопки 0.5×, 0.75×, 1×, 1.25×, 1.5×, 2× через `setPlaybackRate`
- **Полноэкранный режим** — вызывает `containerRef.current.requestFullscreen()` на контейнере плеера. Выход через `document.exitFullscreen()`. Состояние `isFullscreen` синхронизируется через событие `fullscreenchange` на `document`.

**Режим отображения контролов:**
- **Обычный** — контролы расположены под плеером (`position: static`)
- **Полноэкранный** — контролы накладываются поверх плеера (`position: absolute, bottom: 0`) с градиентным фоном. Автоматически скрываются через 3 секунды после последнего движения мыши/касания и показываются снова при активности.


**Автовоспроизведение следующего сегмента в сегментированном видео:**

Маршрут `/segment/:segmentedVideoId/:segmentId` передаёт контекст сегментированного видео в `SegmentPage`. При открытии плеер загружает упорядоченный список сегментов сегментированного видео.

Переход к следующему сегменту (`goToNext`) происходит в двух случаях:

- **`onStateChange: ENDED`** — сегмент досмотрен до конца (для сегментов без ограничения по времени или если пользователь досмотрел дальше `timeEnd`)
- **Достижение `timeEnd`** — если задан временной интервал, `setInterval` (каждые 100ms) проверяет позицию воспроизведения; при `currentTime >= timeEnd - 0.15` вызывается `goToNext`

```
function goToNext():
  currentIndex = segmentedVideoSegments.indexOf(currentSegment)
  nextSegment = segmentedVideoSegments[currentIndex + 1]
  if nextSegment:
    navigate(/segment/{segmentedVideoId}/{nextSegment.uuid})
  else if isLooping:
    // Если включено зацикливание — возвращаемся к первому сегменту
    navigate(/segment/{segmentedVideoId}/{segmentedVideoSegments[0].uuid})
  else:
    navigate(/segmented-video/{segmentedVideoId})
```

Интервал запускается при `PLAYING` и очищается при `PAUSED / BUFFERING / ENDED` и при анмаунте компонента.

Если текущий сегмент последний в сегментированном видео — происходит возврат на страницу сегментированного видео.

**Очистка:**

`cancelAnimationFrame(rafId)` + `player.destroy()` при анмаунте компонента.

**Состояния плеера:**

- `BUFFERING (3)` → спиннер в центре поверх оверлея
- `PLAYING (1)` → спиннер скрыт, иконка паузы скрыта
- `PAUSED (2)` → иконка ▶ в центре

## 9. Создание сегментированного видео (объединено с добавлением сегмента)

Пустые сегментированные видео не допускаются. Создание сегментированного видео всегда требует добавления первого сегмента.

**Маршрут:** `/add` (без параметра `segmentedVideoId`)

Кнопка «Создать сегментированное видео» в `SegmentedVideosPage` ведёт на `/add` (Link, не модал).

### Форма

Форма состоит из двух секций, разделённых визуальным разделителем:

**Секция 1 — Сегментированное видео:**
- Название сегментированного видео — text input, placeholder «Моё сегментированное видео», autofocus

**Секция 2 — Первый сегмент:**
- YouTube URL — text input (парсинг `youtubeId` через regex, debounce 500ms)
- Интерактивный плеер — при валидном `youtubeId` загружается живое видео для предпросмотра и выбора интервала.
- Визуальный выбор интервала — интерактивный слайдер с двумя ползунками под плеером для выбора `timeStart` и `timeEnd`.
- Описание сегмента — textarea (необязательно)
- Временной диапазон — 2 inputs (для точного ввода) прямо под видеоплеером + визуальный слайдер-оверлей поверх плеера. По умолчанию выбирается весь диапазон (0:00 – duration).

### Валидация сегмента через iframe

При нажатии «Создать сегментированное видео»:
1. Валидация полей (название сегментированного видео, URL, диапазон)
2. Создаётся скрытый `YT.Player` (1×1px, off-screen) с указанным `youtubeId`
3. `onReady` → сегмент доступен → сохраняем сегментированное видео + сегмент → переход на `/segmented-video/{segmentedVideoId}`
4. `onError` → ошибка «Видео недоступно или указан неправильный URL»
5. Таймаут 8 секунд → ошибка (сегмент недоступен)

Кнопка во время проверки: disabled, спиннер + текст «Проверяем сегмент…»

## 10. Экран: Добавление сегмента в существующее сегментированное видео (`AddPage`)

**Маршрут:** `/add?segmentedVideoId={uuid}`
**Назначение:** добавление нового сегмента в существующее сегментированное видео

Та же форма что и при создании, но без секции «Название сегментированного видео». Вместо неё — select/дропдаун для выбора сегментированного видео (предзаполнен если передан `segmentedVideoId`).

Сегмент также проходит валидацию через скрытый iframe перед сохранением.

### Поля формы

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| YouTube URL | text input | да | Из URL парсится `youtubeId` через regex `(?:v=\|youtu\.be\/|shorts\/)([a-zA-Z0-9_-]{11})` |
| Сегментированное видео | select / выбор | да | Дропдаун из существующих сегментированных видео. Если открыт из конкретного сегментированного видео — предзаполнен. |
| Описание | textarea | нет | — |
| Временной диапазон | 2 inputs | да | По умолчанию выбирается всё видео |
| Начало сегмента | text input | да | Формат `m:ss`, placeholder `0:00` |
| Конец сегмента | text input | да | Формат `m:ss`, placeholder `duration` |

### Плеер и визуальный выбор интервала

Как только `youtubeId` успешно извлечён — инициализируется `YT.Player`. Под плеером отображается **Range Slider** (двухпозиционный ползунок), позволяющий мышкой/пальцем выбрать границы сегмента.

**Технические детали реализации слайдера:**
- **Адаптивность:** Используется `ResizeObserver` для отслеживания ширины контейнера плеера и переключения между мобильным и десктопным режимами.
- **Точное позиционирование:** Высота слайдера динамически меняется:
    - Десктопная версия: `bottom: 42px`
    - Мобильная версия (< 640px): `bottom: 37px`
- **Синхронизация:** При перемещении ползунков значения в полях «timeStart» и «timeEnd» обновляются автоматически. И наоборот — ручной ввод в поля сдвигает ползунки.

### Валидация

| Поле | Правило | Сообщение |
|---|---|---|
| YouTube URL | Должен распарситься `youtubeId` | «Введите корректную ссылку» |
| Сегментированное видео | Выбран | «Выберите сегментированное видео» |
| timeStart / timeEnd | Оба заполнены | «Заполните оба поля» |
| timeEnd > timeStart | Числовая проверка | «Конец должен быть позже начала» |
| Доступность сегмента | Проверка через скрытый iframe | «Видео недоступно или указан неправильный URL» |

Ошибки — inline, под соответствующим полем. Кнопка «Сохранить» активна всегда, ошибки показываются при сабмите.

### После сохранения

Сегмент добавляется в выбранный сегментированное видео (`segmentedVideo.segmentIds`), данные сохраняются в `localStorage`. Переход на главный экран, сегментированное видео которому добавили сегмент — прокручен в начало, новая карточка видима.

## 11. Экран: Редактирование сегмента (`EditPage`)

Идентичен `AddPage` по структуре, но все поля предзаполнены данными сегмента. 

После сохранения — возврат на `SegmentedVideoPage` с обновлёнными данными.

## 12. Хранилище (`localStorage`)

Все данные хранятся в двух ключах:

```
spark_segmented_videos  →  SegmentedVideo[]   (JSON)
spark_segments  →  Segment[]   (JSON, все сегменты всех сегментированных видео)
```

**Запись:** при каждом изменении (`addSegment`, `editSegment`, `deleteSegment`, `createSegmentedVideo`, `editSegmentedVideo`, `deleteSegmentedVideo`).

**Чтение:** при инициализации приложения (`App` useEffect).

**Обработка ошибок:** `try/catch` при парсинге. Если данные повреждены — инициализация с пустыми массивами, данные не перезаписываются автоматически (нужно явное действие пользователя).

## 15. Известные ограничения и точки роста v2

**Порядок сегментов в сегментированном видео** — в v1 сортировка по дате создания (`createdAt`). В v2 — drag-and-drop сортировка.

**Синхронизация между устройствами** — в v1 только `localStorage`. В v2 — облачный бэкенд.

**Режим изучения** — отдельный режим флэшкарточек по словам сегментированного видео. В v2.

**Autoplay и mute** — браузеры блокируют autoplay со звуком без пользовательского жеста. Первое воспроизведение сегмента всегда без звука.
